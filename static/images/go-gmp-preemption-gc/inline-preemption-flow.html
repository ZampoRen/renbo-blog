检查

是否处在 async safe point？

同步路径：下次函数序言生效
异步路径：进入 asyncPreempt
拒绝位置：runtime / assembly / unsafe point

gopreempt_m

普通抢占：G 回到 runnable，等待再次竞争运行。

preemptPark

停止请求：G 进入 _Gpreempted，等待检查或扫描后恢复。

关键判断：信号只是入口，safe point 才是门槛；抢占请求只是请求，调度完成才算真的让出。
